# Pretty Output - nanobot 输出美化

让 nanobot 的回复更加结构化、美观、易读，适配各类聊天平台。

---

## 效果示例 (Before → After)

用户提问：「Python 列表去重有哪些方法？」

**Before（原始回复）：**

```
Python列表去重方法：
1. 使用set()
2. 使用dict.fromkeys()
3. 使用列表推导式
注意：set()不保留顺序，dict.fromkeys()保留顺序。
```

**After（美化后回复）：**

```
【问题解答】

▎核心回答
Python 列表去重有 3 种常用方法，按推荐程度排列。

▎方法详解
① set() 转换法
   → list(set(data))
   → 最简洁，但不保留原始顺序

② dict.fromkeys() 法
   → list(dict.fromkeys(data))
   → 保留首次出现的顺序 ✅

③ 列表推导式
   → [x for i, x in enumerate(data) if x not in data[:i]]
   → 最灵活，可自定义去重逻辑

💡 推荐：需要保留顺序时用 dict.fromkeys()，不需要时用 set()

───────────
```

## 输出格式规范

### 通用排版规则

1. **标题层级**：用符号区分层级，避免纯 Markdown `#` 标记（部分平台不渲染）
   - 一级标题：`【标题】` 或 `━━ 标题 ━━`
   - 二级标题：`▎标题` 或 `◆ 标题`
   - 三级标题：`▸ 标题`

2. **列表项**：使用可视符号替代 `-`
   - 有序：`①` `②` `③` `④` `⑤` `⑥` `⑦` `⑧` `⑨` `⑩`
   - 无序：`◦` 或 `•` 或 `▪`
   - 勾选：`✅` 完成 / `⬜` 未完成

3. **分隔线**：使用 `───────────` 或 `· · ·` 分隔段落

4. **强调**：
   - 重要：`⚠️ 内容`
   - 提示：`💡 内容`
   - 成功：`✅ 内容`
   - 错误：`❌ 内容`
   - 信息：`ℹ️ 内容`

5. **引用/备注**：用 `「...」` 包裹

## 平台适配指南

### QQ / QQ频道
- 不支持 Markdown 渲染，使用纯文本符号排版
- 避免使用 `**加粗**`、`# 标题` 等 Markdown 语法
- 用全角符号和 emoji 营造视觉层级
- 每段之间保留空行，提升可读性
- 代码段用「代码内容」包裹，不用反引号

### Telegram
- 支持部分 Markdown（加粗、斜体、代码块）
- 可混合使用符号排版 + Markdown
- 代码块可用反引号包裹

### Discord
- 完整支持 Markdown
- 可使用 Markdown + 符号排版结合
- 支持代码块语法高亮

## 使用方式

将此 skill 文件复制到 nanobot 的 skills 目录：

```bash
# 复制到 nanobot 用户 skills 目录
mkdir -p ~/.nanobot/skills
cp SKILL.md ~/.nanobot/skills/pretty-output.md

# 重启 nanobot 生效
pkill -f nanobot
nanobot gateway
```

### 验证 skill 是否生效

① 检查 skill 文件是否已加载：
```bash
ls ~/.nanobot/skills/pretty-output.md
```

② 发送测试消息验证输出格式：
```bash
nanobot agent -m "Python 列表去重有哪些方法？"
```

③ 确认回复中包含格式符号（如 `【】`、`▎`、`①②③`、`───`），即表示 skill 已生效。如果输出仍为普通格式，检查 skill 文件路径和 nanobot 是否已重启。
